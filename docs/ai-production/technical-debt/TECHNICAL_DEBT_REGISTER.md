# Technical Debt Register

This register is the source of truth for known technical, process, asset, QA, marketing, and production debt in the Red Ball AI production workflow.

Every worker final report must state `Technical debt added: <ID/link>` or `Technical debt added: none`. If a shortcut, manual workaround, deferred verification, uncertain source, or temporary process is accepted, it must be recorded here before the task is considered complete.

## Status Values

- Open: known debt exists and has not been reviewed.
- Monitoring: accepted for now, with an explicit review trigger.
- In remediation: active work is reducing or closing the debt.
- Closed: closure criteria are met and evidence is linked.

## How To Add A Debt Entry

1. Copy `docs/ai-production/templates/technical-debt-entry-template.md`.
2. Assign the next `TD-####` ID.
3. Add the row to the register table and link supporting task, report, decision, or asset manifest evidence where practical.
4. Mention the new ID in the worker final report as `Technical debt added: TD-####`.
5. If no debt was added, the worker final report must still say `Technical debt added: none`.
